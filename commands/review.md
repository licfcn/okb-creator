从 wiki 文章自动生成间隔重复复习卡片，用于深度内化知识。

## 功能说明

基于 obsidian-spaced-repetition 插件格式，从 wiki 文章提取核心概念、定义、要点，自动生成问答式复习卡片。

## 前置要求

### 安装 obsidian-spaced-repetition 插件

1. Obsidian 设置 → 第三方插件 → 浏览插件
2. 搜索 "Spaced Repetition"
3. 安装并启用
4. 配置复习间隔（建议：1d, 3d, 7d, 14d, 30d）

### 复习卡片格式

```markdown
#flashcard/tag 
问题内容
?
答案内容
```

## 执行步骤

### 1. 扫描目标文章

根据用户输入确定要生成卡片的文章：

**参数说明**：
- 无参数：提示用户选择
- 文章名：为指定文章生成卡片
- `--type <type>`：为特定类型的所有文章生成
- `--tag <tag>`：为特定标签的所有文章生成
- `--folder <folder>`：为特定文件夹的所有文章生成
- `--all`：为所有 wiki 文章生成卡片（慎用）

### 2. 提取知识点

对每篇文章，提取以下内容生成卡片：

**从概念类文章（concept）**：
- 定义：概念的定义和解释
- 核心要点：关键特征和原则
- 区分：与其他概念的区别
- 应用场景：何时使用

**从框架类文章（framework）**：
- 核心思想：框架的核心理念
- 关键步骤：执行步骤或流程
- 适用场景：何时应用
- 注意事项：常见陷阱

**从工具类文章（tool）**：
- 核心功能：主要功能
- 使用方法：操作步骤
- 最佳实践：使用技巧
- 对比：与其他工具的区别

**从人物类文章（person）**：
- 核心贡献：主要成就
- 关键思想：方法论和理念
- 代表案例：典型实践

### 3. 生成问答卡片

**生成策略**：

1. **定义类问题**：
   ```
   Q: [概念]的定义是什么？
   A: [定义内容]
   ```

2. **要点类问题**：
   ```
   Q: [概念]的核心要点有哪些？
   A: [要点列表]
   ```

3. **应用类问题**：
   ```
   Q: 如何应用[概念]？
   A: [应用步骤或方法]
   ```

4. **对比类问题**：
   ```
   Q: [概念A]和[概念B]的区别是什么？
   A: [对比分析]
   ```

5. **案例类问题**：
   ```
   Q: [概念]的典型应用案例是什么？
   A: [案例描述]
   ```

### 4. 创建卡片文件

**文件命名**：`review/cards/{文章名}-卡片.md`

**文件结构**：

```markdown
---
title: {文章主题} - 复习卡片
source: wiki/{分类}/{文章名}.md
type: {文章type}
created: YYYY-MM-DD
tags: [复习, {原文章tags}]
---

#flashcard/{tag1} 

{问题1}
?
{答案1}

---

#flashcard/{tag2} 

{问题2}
?
{答案2}

---

#flashcard/{tag3} 

{问题3}
?
{答案3}
```

### 5. 更新复习计划

在 `review/schedule.md` 中追加：

```markdown
## YYYY-MM-DD 新增

- [ ] [[{文章名}-卡片]] ({卡片数}张)
```

### 6. 更新统计

在 `review/stats.md` 中更新：

```markdown
## 总览

- 总卡片数：XX（+{新增数}）
- 已复习：XX
- 待复习：XX（+{新增数}）
- 熟练度：XX%
```

### 7. 更新变更日志

在 `wiki/_changelog.md` 顶部追加：

```markdown
## YYYY-MM-DD HH:MM - 生成复习卡片

- 源文章：{文章名}
- 生成卡片：{卡片数} 张
- 保存位置：review/cards/{文章名}-卡片.md
```

## 生成示例

### 示例一：概念类文章

**源文章**：`wiki/思维框架/第一性原理.md`

```markdown
---
topic: 第一性原理
type: concept
---

第一性原理是一种思维方法，要求将问题分解到最基本的事实和假设...

## 定义

第一性原理源自物理学和哲学，指"最基本的、不能从其他命题推导出来的命题"。

## 核心要点

### 1. 分解问题
将复杂问题拆解为最基本单元

### 2. 重新构建
从基本单元出发，寻找新路径

### 3. 避免类比陷阱
不依赖既有经验
```

**生成的卡片**：`review/cards/第一性原理-卡片.md`

```markdown
---
title: 第一性原理 - 复习卡片
source: wiki/思维框架/第一性原理.md
type: concept
created: 2026-06-27
tags: [复习, 思维模型, 推理, 物理学]
---

#flashcard/思维模型 

第一性原理的定义是什么？
?
第一性原理源自物理学和哲学，指"最基本的、不能从其他命题推导出来的命题"。在思维模型中，它代表一种**从零开始推理**的方法论：
- 不依赖类比和既有经验
- 从最基本的事实出发
- 逐步构建复杂结论

---

#flashcard/思维模型 

第一性原理思维的三个核心步骤是什么？
?
1. **分解问题**：将复杂问题拆解为最基本单元
   - 什么是不可改变的事实？
   - 什么是可以改变的假设？
   - 哪些约束是真实的，哪些是想象的？

2. **重新构建**：从基本单元出发，寻找新路径
   - 如果不受现有方案影响，我会怎么做？
   - 有没有更基础的方法解决这个问题？

3. **避免类比陷阱**：不依赖既有经验
   - 避免从众心理
   - 不继承既有方案的缺陷
   - 突破创新限制

---

#flashcard/思维模型 

第一性原理与类比思维的区别是什么？
?
**类比思维**：
- 基于既有经验
- "大家都这么做"
- 容易继承缺陷
- 限制创新空间

**第一性原理**：
- 基于基本事实
- "本质是什么"
- 从零开始构建
- 突破既有框架

---

#flashcard/思维模型 

埃隆·马斯克如何应用第一性原理降低火箭成本？
?
**传统认知**：
- 火箭是一次性使用的
- 成本无法降低

**第一性原理分析**：
- 火箭的材料成本只占总成本的 2%
- 问题在于制造流程，而非材料本身

**解决方案**：
- 可回收火箭
- 成本降低 100 倍
```

### 示例二：框架类文章

**源文章**：`wiki/决策框架/OODA循环.md`

```markdown
---
topic: OODA循环
type: framework
---

OODA循环是一种快速决策框架...

## 关键步骤

1. 观察（Observe）：收集信息
2. 判断（Orient）：分析信息
3. 决策（Decide）：做出选择
4. 行动（Act）：执行决策
```

**生成的卡片**：`review/cards/OODA循环-卡片.md`

```markdown
---
title: OODA循环 - 复习卡片
source: wiki/决策框架/OODA循环.md
type: framework
created: 2026-06-27
tags: [复习, 决策, 框架]
---

#flashcard/决策 

OODA循环的四个步骤是什么？
?
1. **观察（Observe）**：收集信息
   - 环境信息
   - 对手信息
   - 自身状态

2. **判断（Orient）**：分析信息
   - 过滤噪音
   - 识别模式
   - 更新心智模型

3. **决策（Decide）**：做出选择
   - 生成选项
   - 评估选项
   - 选择最优

4. **行动（Act）**：执行决策
   - 快速执行
   - 观察结果
   - 进入下一循环

---

#flashcard/决策 

OODA循环的核心优势是什么？
?
- **快速决策**：缩短决策时间
- **适应变化**：动态调整策略
- **占据主动**：比对手更快完成循环
- **持续优化**：每次循环都改进

---

#flashcard/决策 

如何应用OODA循环？
?
1. **建立信息收集系统**
   - 持续观察环境
   - 多渠道信息源

2. **培养快速判断能力**
   - 预设判断框架
   - 练习模式识别

3. **简化决策流程**
   - 减少决策步骤
   - 设置决策时限

4. **快速执行**
   - 行动优先
   - 边做边调整
```

## 使用示例

```bash
# 为特定文章生成卡片
/review 第一性原理

# 为所有概念类文章生成卡片
/review --type concept

# 为特定标签的文章生成卡片
/review --tag 思维模型

# 为特定文件夹的文章生成卡片
/review --folder 思维框架

# 为所有文章生成卡片（慎用）
/review --all
```

## 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 文章名 | 为指定文章生成卡片 | `/review 第一性原理` |
| `--type <type>` | 为特定类型的所有文章生成 | `/review --type concept` |
| `--tag <tag>` | 为特定标签的所有文章生成 | `/review --tag 思维模型` |
| `--folder <folder>` | 为特定文件夹的所有文章生成 | `/review --folder 思维框架` |
| `--all` | 为所有 wiki 文章生成 | `/review --all` |
| `--force` | 强制重新生成（覆盖已有） | `/review 第一性原理 --force` |

## 复习策略建议

### 每日复习

- 打开 Obsidian
- 使用 Spaced Repetition 插件
- 复习到期的卡片
- 建议时间：10-20 分钟

### 每周回顾

- 查看 `review/stats.md`
- 识别薄弱环节
- 针对性生成更多卡片

### 每月优化

- 分析复习数据
- 调整卡片内容
- 删除过于简单或复杂的卡片

## 注意事项

- 卡片应该是**原子化**的，一个问题对应一个答案
- 答案应该**简洁明了**，避免过长
- 使用**标签**分类，便于分 deck 复习
- 定期**清理**不再需要的卡片
- 对于复杂概念，可以拆分为多张卡片
- 卡片内容应该**忠于原文**，不要编造
