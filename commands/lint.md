扫描 wiki/ 目录中所有 .md 文件，逐一检查格式问题并修复。

## 扫描范围

递归遍历 wiki/ 下所有子文件夹（包括 _MOC/）中的 .md 文件，排除以下文件：
- INDEX.md
- _compile-log.md
- _changelog.md

## 检查项

### 0. 多层级文件夹结构初始化

如果 wiki/ 下没有子文件夹，或大量文章散落在根目录：

1. 读取 INDEX.md 分类结构（不存在则自动生成）
2. 为每个主题创建对应的多层级文件夹（最多 3 层）
3. 根据文章内容和 type 移动到最匹配的文件夹
4. **全局更新所有 `[[]]` 链接**
5. 更新 INDEX.md

### 1. Frontmatter 检查

- 文件是否以 `---` 开头的 YAML frontmatter 开始？
- 是否包含所有必需字段：topic, type, status, freshness, updated, tags, sources？
- type 是否合法：concept | framework | tool | project | person | resource | moc | summary | view？
- status 是否合法：draft | review | stable | compiled？
- freshness 是否合法：evergreen | volatile | time-sensitive？
- updated 日期格式是否为 YYYY-MM-DD？
- tags 是否为数组格式？
- sources 是否为 YAML 块列表格式（`- ` 开头），每项使用 `[[]]` 双链并保留 `.md` 扩展名指向 raw/ 或 research/ 文件？

**修复**：缺失的 frontmatter 根据内容自动补全，格式错误的字段修正。

### 2. 链接格式检查

- "相关概念"、"相关链接"部分的列表项是否使用 `[[文件名]]` 格式？
- 正文中提到其他 wiki 概念时是否用了 `[[]]` 链接？
- 是否有纯文本的概念引用应该改为 `[[]]`？
- MOC 笔记中的所有条目是否都是 `[[]]` 链接？
- 原子笔记是否链接到对应的 MOC？

**修复**：将纯文本引用替换为 `[[]]` 格式。

### 3. 断链检测

- 所有 `[[]]` 链接指向的文件是否存在？
- 如果链接指向不存在的文件，报告并建议创建或修正

**修复**：报告断链，不自动创建文件（需用户确认）。

### 4. 双向链接检查

- A 链接到 B，B 是否也链接到 A？
- 对于重要的概念关联，建议双向链接

**修复**：在目标文章的"相关概念"章节追加反向链接。

### 5. 摘要检查

- frontmatter 之后的第一段是否为 2-3 句话摘要？
- 摘要是否直接开始（不带标题）？

**修复**：缺少摘要则根据内容生成。

### 6. 文件命名检查

- 能用中文表达的普通概念是否错误用了全英文？
- 专有名词保留英文是正确的
- 文件名中是否有空格？（应用连字符 `-`）

**修复**：报告命名不规范的文件，给出建议。用户确认后重命名并全局更新链接。

### 7. 文件夹归属检查

- 文章是否在合理的多层级文件夹中？
- 是否有文章散落在 wiki/ 根目录？
- 文件夹名称是否与 INDEX.md 分类一致？
- 文件夹层级是否超过 3 层？

**修复**：将错误归属的文件移动到正确文件夹，全局更新链接。

### 8. MOC 完整性检查

- `wiki/_MOC/` 下的 MOC 是否链接了对应主题的所有笔记？
- 原子笔记是否在"相关概念"中链接到对应 MOC？
- 顶层 MOC（知识总览.md）是否链接了所有主题 MOC？

**修复**：补充缺失的 MOC 链接。

### 9. Tasks 语法检查

- 任务语法是否正确：`- [ ]` 或 `- [x]`
- 带 `#todo` 标签的任务是否有 `📅` 截止日期？
- 日期格式是否为 `YYYY-MM-DD`？
- emoji 标记是否正确：`📅`(due) `🔼`(scheduled) `🔁`(recurring) `✅`(done)

**修复**：报告语法错误，不自动修改任务内容。

### 10. Dataview 查询检查

- ```dataview 代码块语法是否正确？
- FROM 子句的路径是否用双引号？
- 查询的属性名是否拼写正确？

**修复**：报告语法错误。

## 执行流程

1. 检查 wiki/ 文件夹结构，缺失则初始化
2. 列出 wiki/ 中所有 .md 文件（遍历子文件夹）
3. 逐个检查上述所有项
4. 自动修复可安全修复的问题
5. 对需人工确认的问题（重命名、断链）只报告
6. 输出报告：
   - ✅ 通过检查的文件数量
   - 🔧 自动修复的文件数量和内容
   - 📁 文件夹结构变更
   - ⚠️ 需人工处理的问题
   - 📊 整体合规率
7. 在 `wiki/_changelog.md` 顶部追加 lint 记录
