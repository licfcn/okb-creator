整理今日待办、计划和学习情况，写入当天日记。支持多次执行合并去重。

## 核心原则

- **合并去重**：多次执行 /todo 不会重复追加内容，而是智能合并
- **当天日记**：所有整理结果写入 `daily/YYYY-MM-DD.md`
- **闭环联动**：汇总 wiki 学习、跨笔记任务、知识库建议

## 执行流程

### 1. 确定日期

获取今天的日期（YYYY-MM-DD），并计算本周范围（周一到周日）。

### 2. 读取或创建当天日记

- 检查 `daily/YYYY-MM-DD.md` 是否存在
- **不存在** → 基于 `daily/_template.md` 创建新日记
- **已存在** → 读取已有内容，解析各章节（用 `## ` 标题作为章节分隔）

### 3. 扫描学习情况

扫描 wiki/ 中所有文章的 frontmatter：

- **今日新增**：`updated` 为今天的文章（status 为 draft 或新建）
- **今日更新**：`updated` 为今天且之前已存在的文章
- **近期未复习**：`updated` 超过 7 天的 evergreen 文章、超过 30 天的 volatile 文章

### 4. 扫描任务

扫描整个 vault 中的任务（使用 Tasks 插件语法）：

- **今日到期**：包含 `📅 YYYY-MM-DD`（今天日期）的未完成任务
- **逾期任务**：due 日期早于今天且未完成的任务
- **本周任务**：due 日期在本周范围内的未完成任务
- **无日期任务**：带 `#todo` 标签但无 due 日期的任务

扫描范围：
- `wiki/` 中所有 .md 文件
- `daily/` 中所有 .md 文件
- `outputs/` 中所有 .md 文件

### 5. 生成建议

基于以下信息生成 3-5 条个性化建议：

- **知识缺口**：health-check 中发现的孤立文章、矛盾内容
- **学习节奏**：近期新增文章数量趋势
- **任务负担**：逾期任务数量、本周任务密度
- **复习提醒**：需要复习的文章（根据 freshness 周期）
- **深挖线索**：wiki 中提到但未展开的概念

建议格式：
```
💡 今日建议：
1. [[某概念]] 已经 7 天未复习，建议回顾
2. 发现 3 个逾期任务，建议优先处理 [[某任务]]
3. wiki 中"某方向"材料较少，可以 /research 补充
4. [[概念A]] 和 [[概念B]] 有关联但未链接，建议建立连接
```

### 6. 合并写入日记（关键：去重逻辑）

将扫描结果合并到当天日记。**必须遵守去重规则**：

#### 待办任务合并
- 比对日记中"今日待办"章节已有的任务文本
- 只追加**新发现的**任务，不重复已存在的
- 已完成任务（`- [x]`）保留不动
- 如果任务文本相同但 due 日期变化，更新 due 日期

#### 学习情况合并
- "今日学习"章节**整体替换**为最新扫描结果（因为学习情况是实时快照）
- 保留用户手动添加的学习笔记（如果有）
- 识别用户手动内容：不以 `- [[` 开头的段落视为用户笔记

#### 建议合并
- "学习建议"章节**整体替换**为最新建议（建议应反映当前状态）
- 如果用户在建议下手动写了回复或行动项，保留这些内容

#### 计划合并
- "本周计划"章节：保留已有计划，只追加新计划
- 不修改已完成的计划项

### 7. 更新 Dataview 查询

确保日记中包含以下 Dataview 查询块（如果不存在则添加）：

```markdown
## 📊 任务总览

### 今日到期任务
```dataview
TASK
WHERE !completed AND due = date(this.date)
GROUP BY file.link
```

### 逾期任务
```dataview
TASK
WHERE !completed AND due < date(this.date)
SORT due ASC
```

### 本周任务
```dataview
TASK
WHERE !completed AND due >= date(this.date) AND due <= date(this.date) + dur(7 days)
GROUP BY file.link
```
```

### 8. 输出报告

完成后在终端输出：

```
📅 日记已更新：daily/YYYY-MM-DD.md

📋 任务整理：
- 今日到期：X 个
- 逾期：X 个
- 本周共计：X 个

📖 学习情况：
- 今日新增/更新：X 篇
  - [[文章1]]
  - [[文章2]]
- 待复习：X 篇

💡 已生成 X 条建议

🔗 相关日记：[[昨日日记]]
```

### 9. 记录变更

在 `wiki/_changelog.md` 顶部追加：

```markdown
## YYYY-MM-DD HH:MM - /todo 日记整理

- 日记文件：daily/YYYY-MM-DD.md
- 整理任务：X 个
- 学习记录：X 篇
- 生成建议：X 条
- 操作：合并更新（或新建）
```

## 日记文件结构

日记文件 `daily/YYYY-MM-DD.md` 的标准结构参照 `daily/_template.md`：

```markdown
---
date: YYYY-MM-DD
type: daily
week: 第W周
tags: [daily]
---

# YYYY-MM-DD 周X

## 📋 今日待办

- [ ] 任务描述 📅 YYYY-MM-DD

## 📖 今日学习

### 新增/更新笔记
- [[笔记名]] - 简述

### 待复习
- [[笔记名]] - X天未复习

## 📅 本周计划

- [ ] 计划项 📅 YYYY-MM-DD

## 💡 学习建议

1. 建议一
2. 建议二

## 📊 任务总览

（Dataview 查询块）

## ✏️ 随手记录

（用户自由记录区域）
```

## 去重规则详述

### 任务去重判定
两任务视为重复当且仅当：
- 任务文本完全相同（忽略 emoji 和日期标记）
- 或都链接到同一个 wiki 文章且动作相同

### 多次执行示例

**第一次 /todo（早上）**：
```
今日待办：
- [ ] 阅读[[第一性原理]] 📅 2026-07-03
- [ ] 整理 raw/ 新材料 📅 2026-07-03

学习情况：
- 新增 [[第一性原理]]
```

**第二次 /todo（下午）**：
```
今日待办：
- [ ] 阅读[[第一性原理]] 📅 2026-07-03      ← 保留（已存在）
- [ ] 整理 raw/ 新材料 📅 2026-07-03        ← 保留（已存在）
- [ ] 复习[[OODA循环]] 📅 2026-07-03        ← 新增

学习情况：
- 新增 [[第一性原理]]                        ← 更新（保留）
- 更新 [[思维模型]]                          ← 新增
```

## 注意事项

- 不要删除用户手动添加的内容（随手记录区、手动计划项）
- Dataview 查询块中的 `this.date` 指向日记 frontmatter 中的 date 字段
- 任务语法遵循 Tasks 插件规范：`📅` 表示 due date
- 如果当天是周一，额外生成本周回顾的提示
- 日记中所有 wiki 链接使用 `[[]]` 格式
