整理 wiki/ 中的原子笔记，创建 Map of Content（MOC）和主题总结，建立多层级知识结构。

## 核心理念

这个命令实现 Zettelkasten（卢曼卡片盒）的"永久笔记"层：

- `/compile` 提取原子洞察 → 散落的原子笔记
- `/summarize` 组织原子笔记 → MOC 导航 + 主题总结 + 多层级结构

**Zettelkasten 三层笔记体系**：
1. **Fleeting notes（闪念笔记）** → raw/ 中的随手记录
2. **Literature notes（文献笔记）** → wiki/ 中的原子笔记（/compile 产出）
3. **Permanent notes（永久笔记）** → wiki/_MOC/ 中的 MOC 和主题总结（/summarize 产出）

## 执行流程

### 1. 扫描 wiki/ 全部内容

递归扫描 wiki/ 下所有 .md 文件（排除 `_` 开头的文件）：
- 读取每篇文章的 frontmatter（type, tags, updated, sources）
- 读取每篇文章的 `[[]]` 链接
- 读取每篇文章的摘要（第一段）
- 记录文章所在的多层级文件夹路径

### 2. 分析主题聚类

基于以下维度对原子笔记进行聚类分析：

- **tag 聚类**：共享相同 tag 的文章归为一组
- **链接聚类**：互相链接频繁的文章归为一组
- **type 聚类**：相同 type 的文章归为一组
- **sources 聚类**：来源相同 raw/ 文件的文章归为一组
- **语义聚类**：摘要内容主题相近的文章归为一组

综合以上维度，识别出 3-8 个主要主题簇。

### 3. 组织多层级文件夹结构

将识别出的主题簇组织为多层级文件夹：

```
wiki/
├── _MOC/                          # MOC 导航层
│   ├── 知识总览.md                 # 顶层 MOC（链接所有主题 MOC）
│   ├── 主题A-MOC.md               # 主题 A 的导航
│   └── 主题B-MOC.md               # 主题 B 的导航
├── 主题A/                          # 一级分类
│   ├── 子主题A1/                   # 二级分类
│   │   ├── 原子笔记1.md
│   │   └── 原子笔记2.md
│   ├── 子主题A2/
│   │   └── 原子笔记3.md
│   └── 主题A-总结.md              # 主题 A 的总结笔记
└── 主题B/
    ├── 子主题B1/
    │   └── ...
    └── 主题B-总结.md
```

**多层级规则**：
1. 文件夹最多 3 层深度（`wiki/主题/子主题/` ）
2. 每个叶子文件夹至少有 2 篇文章，否则合并到上级
3. 文件夹名称用中文，专有名词保留英文
4. 当主题内容增多时，可以拆分子主题（动态演化）
5. 移动文件时**全局更新所有受影响的 `[[]]` 链接**

### 4. 创建/更新 MOC 笔记

为每个主题簇创建 MOC（Map of Content）笔记，放在 `wiki/_MOC/`：

**主题 MOC 模板**：

```markdown
---
topic: {主题名称}
type: moc
status: stable
freshness: evergreen
updated: YYYY-MM-DD
tags: [MOC, {主题标签}]
sources:
  - "[[raw/文件名.md]]"  # 本主题下所有文章的 sources 并集，双链格式
---

{主题名称}是当前知识库中的一个核心主题，涵盖以下子方向。

## 📌 核心概念

- [[原子笔记1]] - 一句话概括
- [[原子笔记2]] - 一句话概括
- [[原子笔记3]] - 一句话概括

## 🗂️ 子主题

- [[子主题A1-MOC]] 或链接到子主题总结
- [[子主题A2]]

## 🔗 关联主题

- [[主题B-MOC]] - 关联说明
- [[主题C-MOC]] - 关联说明

## 📖 学习路径

建议阅读顺序：
1. 先读 [[基础概念]] 建立框架
2. 再读 [[进阶概念]] 深入理解

### 5. 创建主题总结笔记

为每个主要主题创建一篇总结笔记，放在主题文件夹根目录：

**主题总结模板**：

```markdown
---
topic: {主题名称}总结
type: summary
status: stable
freshness: volatile
updated: YYYY-MM-DD
tags: [总结, {主题标签}]
sources:
  - "[[raw/文件名.md]]"  # 本主题所有文章的 sources 并集，双链格式

{主题名称}的核心内容整理。本文是对该主题下所有原子笔记的综合梳理，提供整体视角。

### 6. 补充双向链接

扫描所有原子笔记，补充缺失的 `[[]]` 链接：

1. **主题内链接**：同主题的文章如果内容相关但未链接，互相添加 `[[]]`
2. **跨主题链接**：不同主题的文章如果存在概念关联，互相添加 `[[]]`
3. **MOC 反向链接**：每篇原子笔记在"相关概念"部分添加到对应 MOC 的链接
4. **总结链接**：每篇原子笔记添加到主题总结的链接

链接添加规则：
- 在文章末尾"相关概念"章节追加链接
- 如果文章没有"相关概念"章节，创建一个
- 链接格式：`- [[文件名]]`
- 不要在正文中插入过多链接，保持自然

## 与 /compile 的分工

| 命令 | 职责 | 产出 |
|------|------|------|
| /compile | 从 raw/ 提取原子洞察 | wiki/ 中的原子笔记（散落） |
| /summarize | 组织原子笔记为知识结构 | MOC + 总结 + 多层级 + 双链 |

**推荐工作流**：
1. 丢新材料到 raw/
2. `/compile` 提取原子笔记
3. 积累一定量后 `/summarize` 整理结构
4. 继续丢材料、`/compile`
5. 再次 `/summarize` 更新结构

`/summarize` 是幂等的：多次执行不会产生重复的 MOC 或总结，而是更新已有内容。

## 注意事项

- 不要修改原子笔记的核心内容，只补充链接和分类
- MOC 和总结笔记的 `type` 字段分别为 `moc` 和 `summary`
- 移动文件时必须全局更新 `[[]]` 链接
- 文件夹层级不超过 3 层
- 如果主题下文章少于 3 篇，暂不创建独立 MOC，合并到上级
- 尊重用户手动创建的文件夹结构，不强行重组
